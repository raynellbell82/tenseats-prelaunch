**app/launch/checkout/checkout-page-content.tsx** ("use client"):

This component manages the checkout flow:

1. Read `preRegistrationId` from sessionStorage (SSR-safe with useEffect + mounted state, same pattern as QueueStatus component).
2. If no preRegistrationId in sessionStorage, also check URL searchParams for `?preRegistrationId=` (Stripe cancel_url includes this). Use `useSearchParams()` from next/navigation.
3. Query `api.launch.queue.getCheckoutData` with the preRegistrationId (use "skip" pattern when ID is null).
4. Query `api.launch.queue.getLaunchConfig` for `isLifetimeOfferActive`.
5. If checkout data is null (not invited), redirect to /launch.
6. Use `useState` for `selectedTier` initialized from checkout data's tier.
7. Render:
   - Heading: "Complete Your Membership"
   - `<CheckoutCountdown expiresAt={checkoutData.inviteExpiresAt} onExpired={() => router.push("/launch/expired")} />`
   - `<TierSelector selectedTier={selectedTier} onSelect={setSelectedTier} isLifetimeOfferActive={config?.isLifetimeOfferActive ?? false} />`
   - A "Proceed to Payment" button that:
     - Sets loading state
     - Calls `useAction(api.launch.membershipCheckout.createMembershipCheckout)` with `{ preRegistrationId, tier: selectedTier }`
     - On success, redirects browser to the returned `url` via `window.location.href = result.url`
     - On error, shows toast via sonner
   - Show the user's email and category from checkout data as context

8. Loading state: Show skeleton while mounted is false or checkout data is loading.
9. Use max-w-2xl mx-auto px-4 py-12 container layout.

**Important:** Use `useAction` from "convex/react" for `createMembershipCheckout` (it's a Convex action, not a mutation). The import is: `import { useAction } from "convex/react";`
