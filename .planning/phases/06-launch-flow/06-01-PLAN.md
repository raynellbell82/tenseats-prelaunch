Create app/launch/page.tsx as a Server Component (no "use client") that renders the launch flow. Follow the same pattern as app/page.tsx and app/join/page.tsx: Server Component importing and rendering Client Components.

However, this page needs client-side state to coordinate between components (selected city drives slot grid, clicking a slot opens the signup form). Therefore, create a companion "use client" component inline or as a separate wrapper:

**Approach:** Create the page.tsx as a Server Component that renders LandingHeader and a Client Component called LaunchPageContent (defined in the same file or imported). The LaunchPageContent component manages the state machine:

1. Import and render LandingHeader at the top (same as all pages).
2. Create a "use client" LaunchPageContent component (can be in the same file after page export, or defined above) that:
   - Uses `useState` to track `selectedMetroId` and `selectedCategory` (both string | null).
   - Uses `useQuery(api.launch.queue.getLaunchConfig)` to get `isLifetimeOfferActive`.
   - Renders a page title section: "Reserve Your Founding Seat" heading, brief subtext.
   - Renders `<LaunchQueueDialog />` for the "How does this work?" info.
   - Renders `<LaunchCitySearch onSelect={setSelectedMetroId} onClear={() => { setSelectedMetroId(null); setSelectedCategory(null); }} />`.
   - When a city is selected, renders `<SlotGrid selectedMetroId={selectedMetroId} isLifetimeOfferActive={config?.isLifetimeOfferActive ?? false} onJoinQueue={(metroId, category) => { setSelectedMetroId(metroId); setSelectedCategory(category); }} />`.
   - When both metroId and category are selected, renders `<LaunchQueueSignupForm preselectedMetroId={selectedMetroId} preselectedCategory={selectedCategory} />` below the grid.
   - Read `searchParams` for `?city=` parameter (from city pages linking to /launch?city=slug). If present, auto-select that metro. Note: searchParams comes from the Server Component page and should be passed as a prop to LaunchPageContent.

**Layout:** Use a centered container with max-w-4xl mx-auto px-4 py-12 pattern consistent with join page styling. Space sections with gap-8 or space-y-8.

**Important:** The LaunchQueueSignupForm expects `preselectedMetroId` as a string (the Convex metro _id), not a slug. The city search onSelect callback already returns the metro _id.

Since LandingHeader is a Client Component already and the page needs to be a Server Component at the route level, structure as:
```tsx
import { LandingHeader } from "@/components/landing/landing-header";
import { LaunchPageContent } from "./launch-page-content";

export default function LaunchPage() {
  return (
    <>
      <LandingHeader />
      <LaunchPageContent />
    </>
  );
}
```

Then create the LaunchPageContent as a separate "use client" file at app/launch/launch-page-content.tsx. This keeps the page.tsx as a clean Server Component.
