   - Remove `password` from `defaultValues`

   - Remove: `import { signUp } from "@/lib/auth-client";`
   - Add: `import { authClient } from "@/lib/auth-client";`

3. **Rewrite onSubmit to use OTP flow:**
   Instead of `signUp.email({ email, password, name })`, do:
   ```typescript
   // First join the queue (no auth needed to join queue)
   const preRegistrationId = await joinQueue({
     email: values.email,
     metroId: values.metroId as Id<"metros">,
     category: values.category,
   });
   sessionStorage.setItem("preRegistrationId", preRegistrationId);

   // Then send OTP for email verification
   await authClient.emailOtp.sendVerificationOtp({
     email: values.email,
     type: "email-verification",
   });
   sessionStorage.setItem("otp-verify-email", values.email);
   sessionStorage.setItem("otp-sent-ts", Date.now().toString());

   // Redirect to verify-email page (existing OTP entry UI)
   router.push("/verify-email");
   ```

5. **Update UI copy:** Change the account step heading from "Create your account" to "Verify your email" or "Enter your email". The submit button text "Join the Queue" can stay since that's still the action.

Do NOT change the role selection step — it stays as-is. Only the "account" step changes (no password field, OTP-based flow).
