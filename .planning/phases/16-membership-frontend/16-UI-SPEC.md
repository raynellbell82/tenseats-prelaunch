## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(1 0 0) light / oklch(0.145 0 0) dark | Page background, card backgrounds |
| Secondary (30%) | `--card` / `--muted` oklch(0.97 0 0) | Membership status card, payment history rows, section dividers |
| Accent (10%) | `--accent-burnt-sienna` oklch(0.64 0.174 45.9) / #E8723A | Lifetime badge only (Early Bird, Founding), no other elements |
| Destructive | `--destructive` oklch(0.577 0.245 27.325) | No destructive actions in this phase |

Accent reserved for: The lifetime membership badge (`MembershipTierBadge` with `lifetime` variant) displayed on the status card for Early Bird and Founding tier members. Not used for buttons, CTAs, or links.

Tier visual language:
- **Early Bird / Founding (Lifetime):** Burnt sienna accent badge + `ring-2 ring-foreground` card border (matching existing tier-selector selected state pattern).
- **Insider (Subscription):** Default card border, no accent. Secondary/muted surface for status area.
- **Status indicator (active/cancelled/past_due):** Use existing shadcn Badge `default` / `secondary` / `destructive` variants. No custom status colors.

Source: `app/globals.css` CSS variable definitions; `DSGN-01` comment on `--accent-burnt-sienna`; tier-selector.tsx ring pattern.

## Layout Pattern

The `/account/membership` page follows the `PostSignupLayout` pattern established in v1.2:

```
LandingHeader (sticky, full width)
main (flex col, items-center, px-4 py-12 md:py-20)
  div (w-full max-w-2xl mx-auto, space-y-8)
    h1 "Your Membership"  тАФ 28px semibold
    MembershipStatusCard  тАФ full width card
    PaymentHistoryList    тАФ full width, Insider only (conditional render)
```

No motion/framer-motion entrance animations. The membership page is account-functional, not marketing. Keep static render.

The `/launch/success` page receives an additive change only: a secondary link below the existing "Back to Home" button pointing to `/account/membership`.

## Interaction States

### MembershipStatusCard

| State | Visual |
|-------|--------|
| Loading | `Skeleton` blocks matching card content dimensions тАФ tier name row (h-5 w-32), status row (h-4 w-24), button row (h-10 w-36) |
| Loaded тАФ Lifetime | Burnt sienna `MembershipTierBadge`, tier name, "Lifetime Member" label, no billing button |
| Loaded тАФ Insider Active | Default badge, tier name, "Active" status, "Manage Billing" button |
| Loaded тАФ Insider Cancelled | Secondary badge, tier name, "Cancelled" status, "Manage Billing" button (portal still accessible) |
| Error / not authed | Redirect to `/login` тАФ no membership card shown |

### ManageBillingButton

| State | Visual |
|-------|--------|
| Idle | `Button` variant="default", label "Manage Billing", ExternalLink icon (lucide, h-4 w-4) |
| Loading | Loader2 icon spinning, label "OpeningтАж", button disabled |
| Error | Toast via `sonner` тАФ "Could not open billing portal. Try again." |

### PaymentHistoryList

| State | Visual |
|-------|--------|
| Loading | 3 `Skeleton` rows at h-10 full width |
| Empty (Insider, no charges yet) | Empty state: heading "No payments yet", body "Your payment history will appear here after your first charge." |
| Populated | Table with columns: Date, Description, Amount, Status |
| Error | "Could not load payment history. Refresh the page to try again." muted text, no retry button. Muted-text-only treatment is intentional: payment history is supplemental information and a full retry action would be disproportionate to the failure impact. The solution path (refresh) is present in the copy. |

## Lazy Sync Interaction (MEM-05)

On first Insider page load, `syncMyBillingCustomer` fires automatically as a side effect. This is a silent background operation:

- No spinner or UI feedback shown to the user during sync.
- If sync fails, the failure is silent (logged, not surfaced) тАФ the `getMySubscription` query still works independently.
- Sync is gated to Insider tier only. Early Bird and Founding members do not trigger it.
