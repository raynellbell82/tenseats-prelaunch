# Phase 22: Verification Page - Context

**Gathered:** 2026-03-14
**Status:** Ready for planning

<domain>
## Phase Boundary

After queue signup, users land on a warm, branded "almost there" page at `/launch/verify` telling them to check their email for a verification code. Same page for both guest and vendor signups. This page establishes the post-signup design language that carries through Phases 23 (guest success) and 24 (vendor success). Includes creating shared post-signup components for reuse.

</domain>

<decisions>
## Implementation Decisions

### Page mood and visual tone
- Brand-forward bold aesthetic: dark background, strong typography, Tenseats logo prominent — feels premium and exclusive
- Uses the existing site dark theme with LandingHeader at top (consistent with rest of site)
- Green check circle with gentle pulse animation that loops indefinitely — subtle "alive" feeling
- Oversized headline typography (text-4xl+), large breathing room — fewer words, more impact
- This design language becomes the template for all 3 post-signup pages

### Content and messaging
- Brand playful tone: on-brand table metaphors (e.g., "The Table's Almost Set"), slightly playful
- Vertical timeline showing step-by-step flow: 1) Check email 2) Enter code 3) You're in
- Copy generated by tenseats-copy-writer skill during planning — reviewed at checkpoint
- Include a "Didn't get it? Resend code" link below the main message

### Page structure and routing
- Route: `/launch/verify` (new route under /launch)
- Direct access without signup: redirect to `/launch`
- After OTP verification: role-based routing — guest to guest success page, vendor to vendor success page (based on userRoles table)
- Create stub routes for Phases 23/24: `/launch/success/guest` and `/launch/success/vendor` as minimal placeholders until those phases build the real pages
- Routing logic: non-guest role (chef, mixologist, creator, venueHost) -> vendor success; guest -> guest success

### Shared design system
- Create a PostSignupLayout wrapper component for all 3 post-signup pages
- Full design kit: layout + page entrance animation (fade/slide up) + shared typography styles + vertical timeline component + section spacing tokens
- Reusable VerticalTimeline component that takes step data as props — verification page passes email steps, success pages pass onboarding steps
- All shared components live in `components/post-signup/` (new directory)
- Components: post-signup-layout.tsx, vertical-timeline.tsx, plus any supporting pieces

### Claude's Discretion
- Exact green check circle size and glow intensity
- Specific entrance animation timing and easing
- Typography scale details within the oversized constraint
- Spacing tokens and section padding values
- How stub routes look (minimal is fine)

</decisions>

<specifics>
## Specific Ideas

- Green check circle: solid circle with checkmark icon, gentle scale + glow pulse that loops forever
- Vertical timeline for steps — not numbered list, an actual visual timeline with dots/lines connecting steps
- "The Table's Almost Set" or similar table metaphor for headline (copy-writer skill to finalize)
- Dark premium feel like an invite confirmation — "you just got into something special"
- Resend code link for practical UX

</specifics>
