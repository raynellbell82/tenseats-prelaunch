## Critical Gaps

### 1. initialCitySlug Not Applied (LNCH-01, CITY-06)

**Location:** `app/launch/launch-page-content.tsx`

**Problem:** `LaunchPageContent` receives `initialCitySlug` prop (line 15) but `selectedMetroId` is always initialized to `null` (line 16). No `useEffect` or initializer reads from the prop.

**Impact:** When a visitor clicks "Reserve My Seat" on any city page (`/cities/chicago-il` → `/launch?city=chicago-il`), the city search box is blank and the slot grid is hidden. The user must manually re-search for the city.

**Fix:** Add a `useEffect` that resolves `initialCitySlug` to a Convex metro ID and sets `selectedMetroId`, or pass slug directly to `LaunchCitySearch` as a default value.
