## Critical Gap

### Queue Signup → Email Verification Flow (LNCH-03, LNCH-04, AUTH-02)

**Location:** `components/launch/launch-queue-signup-form.tsx` → `app/(auth)/verify-email/page.tsx`

**Problem:** Phase 11 correctly rewired the queue signup form to use OTP (`authClient.emailOtp.sendVerificationOtp`) instead of password-based auth. However, the form redirects to `/verify-email` after sending the OTP. The `/verify-email` page (from Phase 5) checks `session.user` on mount — if no auth session exists, it redirects to `/login`. Queue users who haven't signed up yet have no auth session, so they get bounced to `/login` before they can enter the OTP code.

**Impact:** The queue signup → email verification path is broken for new users who aren't already authenticated.

**Fix options:**
- A) Create a dedicated `/launch/verify` page that accepts OTP without requiring an auth session
- B) Modify `/verify-email` to detect queue context from sessionStorage and skip the session guard
- C) Have the queue form redirect to `/signup` pre-filled with the user's email instead
